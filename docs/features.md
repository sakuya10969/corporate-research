# 機能一覧

ユーザーが利用できる機能の一覧。
各機能の技術的な仕様・実装詳細は `specs.md` を、API定義は `api-design.md` を参照。

---

## 実装済み機能

### F-001: 企業URL入力

企業の公式サイトURLを入力して分析を開始する。

- テキストフィールドにURL（例: `https://www.toyota.co.jp/`）を入力し、「分析する」ボタンを押す
- `http://` または `https://` で始まるURLのみ受け付ける
- 未入力・不正な形式の場合はエラーメッセージが表示される
- 分析中はフォームが無効化され、二重送信できない

### F-002: 企業分析実行

入力されたURLをもとに、企業の公開情報を自動収集・AI分析する。

- URLを送信すると、企業サイトの情報収集が自動で始まる
- 収集した情報をAIが構造化・要約し、分析結果を返す
- 収集対象は入力URLのドメイン内の公開ページのみ（最大15ページ）
- 情報が取得できなかった項目は表示されない

#### 分析で得られる情報

| 項目 | 内容 |
|------|------|
| 企業プロフィール | 社名、設立年、代表者、所在地、従業員数、資本金 |
| 事業領域 | 企業が展開している事業分野 |
| プロダクト・サービス | 提供している製品・サービスの一覧 |
| 財務情報 | 売上高、営業利益、純利益、成長率 |
| ニュース | 最近のプレスリリース・ニュース |
| リスク要因 | 事業上のリスク |
| 企業概要サマリー | AIによる企業の概要説明 |
| 事業モデル | AIによる事業モデルの説明 |
| SWOT分析 | 強み・弱み・機会・脅威 |
| 競合企業（推定） | AIが推定する競合企業 |
| 今後の展望 | AIによる将来展望の考察 |

### F-003: 分析結果表示

分析結果をセクションごとに整理して表示する。

- 企業名とURLをヘッダーに表示
- 各分析項目をカード形式で表示（企業概要、プロフィール、事業モデル、財務情報、SWOT分析、リスク、競合、展望、ニュース、参照ソース）
- 参照ソースはカテゴリバッジ付きリンクで表示
- 分析中はスケルトンUIが表示される
- エラー発生時はエラーメッセージが表示される
- データが存在しないセクションは非表示

### F-004: Markdownレポート出力

分析結果をMarkdown形式のレポートとして確認できる。

- 分析結果ページ下部の「レポート全文を表示」をクリックすると展開される
- テキストをコピーして外部ツールに貼り付けて利用できる
- レポートには全分析項目と参照ソースが含まれる

### F-005: 企業データ永続化

分析した企業の情報をDBに保存し、再利用できる。

- 同じ企業URLを入力すると、過去に分析済みであることが通知される
- 過去の分析結果をそのまま表示するか、最新情報で更新するかを選択できる
- 初回分析・再分析ともに結果はDBに保存される

### F-006: 差分更新分析

同じ企業を再分析する際、前回からの変更点を差分レポートとして表示する。

- 保存済み企業のURLを入力（または「更新」ボタンを押す）と差分分析が実行される
- 前回分析からの変更点が強調表示される
- 変更がなかった場合は「変更なし」と表示される
- 差分レポートには、新しいニュース・プロフィール変更・新プロダクト・財務変化が含まれる

### F-007: 深掘り分析

保存済みの企業データをもとに、追加の質問で詳細な分析を得られる。

- 分析結果ページの「深掘り分析」タブに質問入力欄が表示される
- 自由文で質問を入力して送信すると、AIが保存済みデータをもとに回答する
- 質問と回答の履歴が会話形式で表示される
- 保存済みデータがない企業には利用不可

#### 質問例

- 「この会社の強みの本質は何か」
- 「競合他社と比べて何が違うか」
- 「成長性の根拠となる情報はあるか」
- 「就活生として注目すべきポイントは何か」

### F-008: 分析履歴管理

企業ごとの分析実行履歴を一覧で確認できる。

- 分析結果ページの「分析履歴」タブに過去の実行一覧が表示される
- 実行日時・種別（初回分析 / 差分更新 / 深掘り）・状態（完了・エラー・実行中）が確認できる
- 過去の分析結果を選択して内容を確認できる

### F-009: ダウンロード機能

分析結果をPDFまたはWord形式でダウンロードできる。

- 分析結果ページの「ダウンロード」ボタンからフォーマットを選択する（PDF / Word）
- サーバーがファイルを生成してブラウザにダウンロードされる
- ファイルには全分析項目と参照ソースが含まれる
- ファイル名: `{企業名}_{YYYY-MM-DD}.pdf` / `{企業名}_{YYYY-MM-DD}.docx`

### F-010: 分析テンプレート（目的別プリセット）

分析の目的に応じたプリセットを選択し、重点的に分析する観点を変えられる。

- 企業URL入力フォームの下に「分析の目的」セレクターが表示される
- 目的を選択して分析を実行すると、選択した観点で結果が強調・整理される
- 未選択時は「総合分析」として処理される

#### テンプレート一覧

| テンプレート | 主な用途 | 重点観点 |
|------------|---------|---------|
| 総合分析（デフォルト） | 汎用 | 全項目均等 |
| 就活・転職リサーチ | 就活生・転職者 | 企業文化・採用情報・成長性・安定性 |
| 投資リサーチ | 個人投資家・アナリスト | 財務情報・成長率・競合優位性・リスク |
| 競合調査 | 事業開発・営業 | 事業領域・プロダクト・差別化要因・市場ポジション |
| 提携先調査 | BD・アライアンス担当 | 事業シナジー・技術力・財務健全性 |

### F-011: 企業名検索（URL補完）

企業名を入力するだけで公式サイトURLを自動補完できる。

- 入力フォームに企業名（例: 「トヨタ」「Sony」）を入力すると候補が最大5件表示される
- 候補を選択するとURLフィールドに自動入力される
- 検索結果が0件の場合は「見つかりませんでした。URLを直接入力してください」と表示される
- 候補はユーザーが確認・選択してから分析を開始する（自動実行しない）

### F-012: 分析結果シェア

分析結果を固有URLで共有できる。

- 分析結果ページの「シェア」ボタンを押すと共有用URLがクリップボードにコピーされる
- URLを知っている人なら誰でもブラウザで閲覧できる（認証不要）
- 共有ページは読み取り専用で、全セクションと分析日時が表示される
- SNSシェア時にはOGPプレビューが表示される

### F-013: 複数企業比較

2〜3社の企業を並べて比較分析できる。

- トップページの「複数企業を比較する →」リンクから比較モードに切り替える
- 企業URL入力フィールドが最大3つ表示される
- 「比較分析する」ボタンで各企業の分析を並行実行する
- 結果は横並びの比較テーブルで表示される（プロフィール、事業領域、財務情報、SWOT分析）
- AIによる比較サマリーが生成される

### F-014: 分析スコアリング

分析結果を複数の観点で数値スコア化し、強み・弱みを直感的に把握できる。

- 分析結果ページの上部に「スコアサマリー」が表示される
- 各観点のスコア（0〜100点）と根拠テキストが表示される
- スコアはLLMによる推定値であり、客観的な評価ではない旨が明示される

#### スコア観点

| 観点 | 説明 |
|------|------|
| 財務健全性 | 売上・利益・成長率の安定度 |
| 成長性 | 事業拡大・新規領域への展開度 |
| 競合優位性 | 差別化要因・市場ポジション |
| リスク度 | 事業リスクの多さ・深刻度 |
| 情報透明性 | 公開情報の充実度 |

### F-015: ユーザー認証（Google OAuth）

Clerk を利用した Google OAuth 認証でログイン・サインアップできる。

- サインイン / サインアップページが用意されている
- Google アカウントでログインできる
- ログイン後、ユーザー情報がアプリケーションDBに同期される
- 未認証で保護ルートにアクセスするとサインインページにリダイレクトされる

## 未実装機能（将来）

### RAG統合

社内ナレッジとの統合により、企業分析に社内情報を加味した回答を得られるようにする。

- 保存済みの社内ドキュメントやナレッジベースを分析コンテキストに追加
- 深掘り質問時に社内情報も参照して回答を生成
