# プロジェクト概要

## プロダクト名

企業分析エージェント（Corporate Research Agent）

## 目的

企業名を入力するだけで、その企業に関する公開情報を自動収集・整理・要約し、構造化された分析結果を返す業務支援プロダクト。

「AIと会話する」チャットUIではなく、「企業分析という業務を自動化する」タスク完了型プロダクトとして設計する。

## 解決したい課題

- 企業調査に必要な情報が複数のWebソースに散在しており、手動での収集・整理に時間がかかる
- 収集した情報を構造化・要約するには一定のスキルと時間が必要
- 調査のたびに同じ作業を繰り返す非効率さ

## 想定ユーザー

| ユーザー | ユースケース |
|---------|------------|
| 営業担当者 | 商談前の企業リサーチ |
| 経営企画・事業開発担当者 | 市場調査・競合分析の起点 |
| 投資・M&A関連の調査担当者 | 投資先・買収先の事前調査 |
| コンサルタント | クライアント企業の事前調査 |

## MVPスコープ

### 対象機能

1. 企業名入力フォーム
2. バックエンドでのWeb情報収集（httpx + BeautifulSoup）
3. 収集情報の整理・構造化
4. Azure AI Foundry 経由での AI 要約・分析
5. フロントエンドでの分析結果表示

### MVP対象外

- ユーザー認証・認可
- 分析履歴の保存・管理
- 分析テンプレート機能
- 複数企業の比較分析
- 調査対象のブックマーク・保存
- LangGraph によるマルチステップワークフロー
- RAG（検索拡張生成）

## 将来的な拡張構想

| フェーズ | 機能 |
|---------|------|
| Phase 2 | 分析履歴管理、結果の保存・再閲覧 |
| Phase 3 | 分析テンプレート（業界別・目的別の分析観点プリセット） |
| Phase 4 | 複数企業の比較分析 |
| Phase 5 | LangGraph を用いた複雑なマルチステップワークフロー |
| Phase 6 | RAG 導入による社内ナレッジとの統合 |

## チャットUIとの違い

| 観点 | チャットUI | 企業分析エージェント |
|------|-----------|-------------------|
| 入力 | 自由文 | 企業名（構造化入力） |
| 処理 | 汎用的なLLM応答 | 定型の分析フロー（収集→整理→分析） |
| 出力 | テキスト応答 | 構造化された分析レポート |
| 体験 | 対話的 | タスク完了型 |
| 価値 | 汎用的な回答 | 業務に直結する分析結果 |
