# デザインシステム — 企業分析エージェント

## デザインコンセプト

業務支援プロダクトとしての信頼感と視認性を重視した、クリーンで落ち着いたデザイン。
白を基調とした明るい画面に、青のアクセントで操作対象を明確にする。
装飾を抑え、情報の構造と読みやすさを最優先する。

## カラーパレット

### 基本方針

- 背景は白。コンポーネント（ボタン、アクティブ要素、強調表示）は青で統一する
- 水色ではなく、はっきりとした青を使う
- 青の使用は操作可能な要素・重要な情報に限定し、視線誘導を明確にする

### カラー定義

| 名前 | 用途 | カラー |
|------|------|--------|
| Primary | ボタン、リンク、アクティブ状態、重要な強調要素 | #2563EB（Blue 600） |
| Primary Hover | ボタンホバー、リンクホバー | #1D4ED8（Blue 700） |
| Primary Light | 選択状態の背景、バッジ背景、軽い強調 | #DBEAFE（Blue 100） |
| Background | ページ全体の背景 | #FFFFFF（白） |
| Surface | カード、セクション、入力フィールドの背景 | #F8FAFC（Slate 50） |
| Border | カード境界線、入力フィールド境界線、区切り線 | #E2E8F0（Slate 200） |
| Text Primary | 本文、見出し | #1E293B（Slate 800） |
| Text Secondary | 補足テキスト、ラベル、キャプション | #64748B（Slate 500） |
| Error | バリデーションエラー、エラーメッセージ | #DC2626（Red 600） |
| Error Light | エラー背景 | #FEF2F2（Red 50） |
| Success | 成功状態、完了表示 | #16A34A（Green 600） |

### 色の使い分けルール

- 青は「操作できるもの」「注目すべきもの」にのみ使う
- 情報表示のみの要素には青を使わない（テキストはSlate系で統一）
- エラーは赤、成功は緑。青と混同しない色を使う
- グレー系はSlateを採用し、温かみのある中間色で統一する

## タイポグラフィ

### フォント

- 日本語・英語混在を想定し、システムフォントスタックを基本とする
- Mantine のデフォルトフォント設定を活用する

### サイズ体系

| 用途 | サイズ | ウェイト |
|------|--------|---------|
| ページタイトル | 24px | Bold（700） |
| セクション見出し | 18px | SemiBold（600） |
| カード見出し | 16px | SemiBold（600） |
| 本文 | 14px | Regular（400） |
| 補足テキスト | 13px | Regular（400） |
| ラベル・キャプション | 12px | Medium（500） |

### テキストカラーの使い分け

- 見出し・本文: Text Primary（#1E293B）
- 補足・ラベル: Text Secondary（#64748B）
- リンク・操作テキスト: Primary（#2563EB）
- エラーメッセージ: Error（#DC2626）

## コンポーネントスタイル

### ボタン

- プライマリボタン: 青背景（#2563EB）+ 白テキスト。角丸 8px
- ホバー時: やや濃い青（#1D4ED8）に変化
- セカンダリボタン: 白背景 + 青ボーダー + 青テキスト
- 無効状態: 背景・テキストともにグレーアウト。操作不可であることを明示
- ローディング状態: ボタン内にスピナーを表示し、テキストは維持

### 入力フィールド

- 白背景、Slate 200 のボーダー（1px）
- フォーカス時: 青ボーダー（#2563EB）に変化。影は控えめ
- エラー時: 赤ボーダー + フィールド下にエラーメッセージ（赤テキスト）
- ラベルは入力フィールドの上に配置（フローティングラベルは使わない）

### カード

- 白背景（#FFFFFF）またはSurface（#F8FAFC）
- Slate 200 のボーダー（1px）で区切る
- 影（box-shadow）は使わないか、ごく控えめにする
- 角丸 8px
- カード内のパディングは 16〜24px

### スケルトンUI（ローディング）

- 分析結果の読み込み中はスケルトンUIを表示する
- スケルトンの色は Surface（#F8FAFC）をベースに、微かなアニメーションで読み込み中を示す
- 実際のコンテンツと同じレイアウト構造を維持する

### バッジ・タグ

- 青系: Primary Light 背景（#DBEAFE）+ Primary テキスト（#2563EB）
- 角丸は完全な丸（pill 型）

### アイコン

- Tabler Icons を使用
- サイズは本文に合わせて 16〜20px
- 色はテキストと同じ色を使う（青アイコンは操作可能な要素のみ）

## レイアウト方針

### 全体構成

- ページ中央寄せ。最大幅 960px 程度
- 左右に十分な余白を確保し、情報が詰まりすぎない印象にする
- ヘッダーはシンプルに。プロダクト名のみ。ナビゲーションはMVPでは不要

### 企業名入力画面

- ページ中央に入力フォームを配置
- 入力フィールド + 送信ボタンを横並びまたは縦並びで配置
- プロダクトの目的を簡潔に示すキャッチコピーを入力フォームの上に配置
- 余計な装飾は不要。入力に集中できるミニマルな構成

### 分析結果画面

- 企業名を大きく表示（ページタイトル）
- 以下のセクションをカードで区切って縦に並べる：
  - 企業概要（summary）
  - 事業内容（business_description）
  - 主要な発見事項（key_findings）— リスト形式
  - 参照ソース（sources）— リンク付きリスト
- 各セクションの見出しは SemiBold で明確に区別する
- セクション間の余白は 24px 程度

### エラー表示

- エラーメッセージはページ上部またはフォーム直下に表示
- Error Light 背景 + Error テキストで目立たせる
- アイコン（警告マーク）を添える

## インタラクション

### 状態遷移

| 状態 | 表示 |
|------|------|
| 初期状態 | 企業名入力フォーム（空） |
| 入力中 | フォームにテキスト入力。バリデーションエラーがあればリアルタイム表示 |
| 送信中 | ボタンがローディング状態。入力フィールドは無効化。スケルトンUIで結果エリアを表示 |
| 結果表示 | 分析結果をカード形式で表示 |
| エラー | エラーメッセージを表示。再試行可能 |

### アニメーション

- 過度なアニメーションは使わない
- 状態遷移（ローディング→結果表示）はフェードインで自然に切り替える
- スケルトンUIのパルスアニメーションのみ許容

## Do's and Don'ts

### Do

- 青は操作可能な要素にのみ使う
- 白背景を基調とし、情報の視認性を最優先する
- カード間・セクション間に十分な余白を取る
- テキストのコントラスト比 4.5:1 以上を維持する
- 日本語テキストの行間は 1.7〜1.8 を確保する
- ローディング状態を必ず表示し、ユーザーを待たせない印象を作る

### Don't

- 水色やパステルブルーを使わない。はっきりとした青（#2563EB）を使う
- 影（box-shadow）を多用しない。ボーダーで区切る
- グラデーションを使わない
- 角丸のサイズを混在させない（8px で統一）
- 装飾的なイラストやアニメーションを入れない
- 情報密度を上げすぎない。業務ツールだが、読みやすさを犠牲にしない

## Mantine テーマとの対応

- Mantine の `primaryColor` を `blue` に設定し、shade 6（#2563EB 相当）をデフォルトにする
- `defaultRadius` を `md`（8px）に設定
- カラースキームは `light` 固定（ダークモードはMVPでは対応しない）
- コンポーネントのデフォルトスタイルは Mantine のテーマオーバーライドで統一する
