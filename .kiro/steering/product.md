# プロダクト概要 — 企業分析エージェント

## 目的

企業URLを入力するだけで、その企業サイトから公開情報を自動収集・整理・要約し、構造化された分析結果を返す業務支援プロダクト。

単なるチャットUIではなく、「企業分析」という明確な業務フローを持つプロダクトとして設計する。ユーザーは自由に質問するのではなく、企業URLを入力し、定型の分析フローに沿って結果を受け取る。Google検索は使用せず、企業サイトの一次情報のみを基に分析する。

## 解決したい課題

- 企業調査に必要な情報が複数のWebソースに散在しており、手動での収集・整理に時間がかかる
- 収集した情報を構造化・要約するには一定のスキルと時間が必要
- 調査のたびに同じ作業を繰り返す非効率さ

## 想定ユーザー

- 営業担当者（商談前の企業リサーチ）
- 経営企画・事業開発担当者（市場調査・競合分析の起点）
- 投資・M&A関連の調査担当者
- コンサルタント（クライアント企業の事前調査）

## MVPスコープ

MVP では以下の機能に絞る：

1. 企業URL入力フォーム
2. バックエンドでのサイト情報収集（httpx + BeautifulSoup、サイトマップ探索・内部リンク探索）
3. 収集情報の整理・構造化・ページ分類
4. Azure AI Foundry 経由での AI 構造化抽出・要約・SWOT分析
5. Markdown レポート・差分検知レポート生成
6. フロントエンドでの分析結果表示

MVP で対象外とするもの：

- ユーザー認証・認可
- 分析履歴の保存・管理
- 分析テンプレート機能
- 複数企業の比較分析
- 調査対象のブックマーク・保存
- LangGraph によるマルチステップワークフロー
- RAG（検索拡張生成）

## プロダクトの定義 — チャットUIとの違い

| 観点 | チャットUI | 企業分析エージェント |
|------|-----------|-------------------|
| 入力 | 自由文 | 企業URL（構造化入力） |
| 処理 | 汎用的なLLM応答 | 定型の分析フロー（サイト収集→整理→構造化抽出→要約） |
| 出力 | テキスト応答 | 構造化された分析レポート |
| 体験 | 対話的 | タスク完了型 |
| 価値 | 汎用的な回答 | 業務に直結する分析結果 |

このプロダクトは「AIと会話する」ものではなく、「企業分析という業務を自動化する」ものである。
